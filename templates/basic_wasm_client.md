# Basic WebAssembly Client

This template is auto-generated by `github.com/tinywasm/client` when `web/client.go` does not exist
in a Go module. It serves as both the WASM entry point and as inline documentation for the
tinywasm ecosystem.

## How it fits in the ecosystem

- **`github.com/tinywasm/app`** — orchestrates everything: in-memory HTTP server (port 6060),
  MCP server (for LLM interaction), browser handler, and other subsystem managers.
- **`github.com/tinywasm/client`** — detects this file, compiles it to WASM, and serves it.
  Three compilation modes are available via the DevTUI (keyboard shortcuts L / M / S):
  - `L` Large  — standard Go compiler, fastest build, largest binary (development default).
  - `M` Medium — TinyGo `-opt=1`, debug-friendly, smaller binary.
  - `S` Small  — TinyGo `-opt=z`, maximum size reduction (production).
- **`github.com/tinywasm/dom`** — the **only** DOM access library in the ecosystem.
  Do NOT use `syscall/js` directly; use `tinywasm/dom` instead.

## Standard library replacements (required for TinyGo / M and S modes)

Never import Go stdlib in WASM files. Use the tinywasm equivalents:

| Stdlib                                   | Replacement                          | Docs                                              |
|------------------------------------------|--------------------------------------|---------------------------------------------------|
| `fmt`, `errors`, `strconv`, `strings`    | `github.com/tinywasm/fmt`            | https://pkg.go.dev/github.com/tinywasm/fmt        |
| `encoding/json`                          | `github.com/tinywasm/json`           | https://pkg.go.dev/github.com/tinywasm/json       |
| `time`                                   | `github.com/tinywasm/time`           | https://pkg.go.dev/github.com/tinywasm/time       |
| `net/http`                               | `github.com/tinywasm/fetch`          | https://pkg.go.dev/github.com/tinywasm/fetch      |
