# Basic WebAssembly Client

This template is auto-generated by `github.com/tinywasm/client` when `web/client.go` does not exist
in a Go module. It serves as both the WASM entry point and as inline documentation for the
tinywasm ecosystem.

## How it fits in the ecosystem

- **`github.com/tinywasm/app`** — orchestrates everything: in-memory HTTP server (port 6060),
  MCP server (for LLM interaction), browser handler, and other subsystem managers.
- **`github.com/tinywasm/client`** — detects this file, compiles it to WASM, and serves it.
  Three compilation modes are available via the DevTUI (keyboard shortcuts L / M / S):
  - `L` Large  — standard Go compiler, fastest build, largest binary (development default).
  - `M` Medium — TinyGo `-opt=1`, debug-friendly, smaller binary.
  - `S` Small  — TinyGo `-opt=z`, maximum size reduction (production).
- **`github.com/tinywasm/dom`** — the **only** DOM access library in the ecosystem.
  Do NOT use `syscall/js` directly; use `tinywasm/dom` instead.

## Standard library replacements (required for TinyGo / M and S modes)

Never import Go stdlib in WASM files. Use the tinywasm equivalents:

| Stdlib                                   | Replacement                          | Docs                                              |
|------------------------------------------|--------------------------------------|---------------------------------------------------|
| `fmt`, `errors`, `strconv`, `strings`    | `github.com/tinywasm/fmt`            | https://pkg.go.dev/github.com/tinywasm/fmt        |
| `encoding/json`                          | `github.com/tinywasm/json`           | https://pkg.go.dev/github.com/tinywasm/json       |
| `time`                                   | `github.com/tinywasm/time`           | https://pkg.go.dev/github.com/tinywasm/time       |
| `net/http`                               | `github.com/tinywasm/fetch`          | https://pkg.go.dev/github.com/tinywasm/fetch      |

//go:build wasm

// web/client.go — WebAssembly entry point for this module.
// Auto-generated by tinywasm when this file does not exist.
// Compiled to web/public/client.wasm and served on port 6060.
//
// DOM access: use github.com/tinywasm/dom — https://pkg.go.dev/github.com/tinywasm/dom
// Do NOT use syscall/js directly; tinywasm/dom wraps it with a builder API.
//
// Compilation modes (L/M/S) are managed by tinywasm — https://pkg.go.dev/github.com/tinywasm/app
// For M and S modes (TinyGo): avoid all Go stdlib — use tinywasm/* replacements (see table above).

package main

import (
	. "github.com/tinywasm/dom" // https://pkg.go.dev/github.com/tinywasm/dom
)

// App holds application state. Embed dom.Element as a value (never pointer) to
// minimize heap allocations in TinyGo's GC.
type App struct {
	Element
	clicks int
}

func (a *App) Render() *Element {
	btnClass := "btn"
	if a.clicks > 0 {
		btnClass = "btn clicked"
	}

	return Div(
		H1("Hello from tinywasm!"),
		Button("Click me").Class(btnClass).On("click", func(e Event) {
			a.clicks++
			a.Update() // re-renders only this component
		}),
		P("Clicks: ", a.clicks),
	)
}

func main() {
	// Inline CSS is fine for prototyping, but in a real component move styles to
	// ssr.go (//go:build !wasm) embedded with //go:embed — keeps them out of the
	// WASM binary. See: https://pkg.go.dev/github.com/tinywasm/components
	Append("head", Style(`
		.btn { padding: .4rem 1rem; cursor: pointer; border: none; border-radius: 4px; background: #007bff; color: white; }
		.btn.clicked { background: #28a745; }
	`))

	Render("app", &App{})

	// select{} keeps the WASM goroutine alive so JS event callbacks keep working.
	select {}
}
